3. ОПИСАНИЕ КЛАССОВ
================================================================================

3.1. Device (Device.cs) - Базовый абстрактный класс
-------------------------------------------------------------------------------
Поля (4):
  - _name          string    - Название устройства
  - _manufacturer  string    - Производитель
  - _isOn          bool      - Состояние (вкл/выкл)
  - _createdAt    DateTime   - Дата создания

Геттеры/Сеттеры:
  + Name, Manufacturer, IsOn, CreatedAt

Конструкторы (3):
  - Device()                         - по умолчанию
  - Device(name, manufacturer)       - с параметрами
  - Device(other)                    - копирования

Методы:
  - TurnOn()      virtual   - Включить устройство
  - TurnOff()     virtual   - Выключить устройство
  - GetStatus()   virtual   - Получить статус
  - GetDeviceType() virtual  - Получить тип
  - GetDetailedInfo() virtual- Получить детали
  - GetShortDescription() abstract - ОБЯЗАТЕЛЕН для наследников


3.2. Thermostat (Thermostat.cs) - Наследник Device
-------------------------------------------------------------------------------
Собственные поля (4):
  - _currentTemperature  double  - Текущая температура
  - _targetTemperature   double  - Целевая температура
  - _mode               string  - Режим (heat/cool/auto)
  - _humidity          double  - Влажность

Конструкторы (4):
  - Thermostat()                           - по умолчанию
  - Thermostat(name, manufacturer, temp)   - основной
  - Thermostat(...полные параметры...)     - полный
  - Thermostat(other)                      - копирования

Собственные методы (4):
  - SetTemperature(temp)       - установить температуру
  - SetMode(mode)              - изменить режим
  - ReadCurrentTemperature()   - прочитать температуру
  - GetClimateInfo()           - получить климат-инфо

Перекрытые методы (override):
  - TurnOn()         - дополнен запуском режима
  - GetDeviceType()  - возвращает "Thermostat"
  - GetDetailedInfo()- детальная информация
  - GetShortDescription() - краткое описание


3.3. Light (Light.cs) - Наследник Device
-------------------------------------------------------------------------------
Собственные поля (4):
  - _brightness     int   - Яркость (0-100)
  - _color         string- Цвет
  - _autoMode      bool  - Авторежим
  - _scheduledTime int   - Время включения

Собственные методы (4):
  - SetBrightness(level)     - установить яркость
  - SetColor(color)          - изменить цвет
  - EnableAutoMode(enable)   - авт. включение
  - GetLightInfo()          - световая информация

Перекрытые методы:
  - TurnOn(), GetDeviceType(), GetDetailedInfo(), GetShortDescription()


3.4. DoorLock (DoorLock.cs) - Наследник Device
-------------------------------------------------------------------------------
Собственные поля (5):
  - _isLocked       bool   - Состояние (заперт/открыт)
  - _accessCode    string - Код доступа
  - _wrongAttempts int    - Неверные попытки
  - _autoLock      bool   - Автоблокировка
  - _lastAccessTime string- Время последнего доступа

Собственные методы (5):
  - TryUnlock(code)    - попытка открыть

  - EnableAutoLock()  - установить блокировку
  - GetLockInfo()     - информация о замке

Перекрытые методы:

Перекрытые методы:

4. БАЗА ДАННЫХ (SQLite)
================================================================================
Файл: smart_home.db (создается автоматически)

Таблицы:

  Таблица Devices (общие данные):
    Id              INTEGER PK AUTOINCREMENT
    Name            TEXT NOT NULL
    Manufacturer    TEXT NOT NULL
    IsOn            INTEGER DEFAULT 0
    DeviceType      TEXT NOT NULL
    CreatedAt       TEXT NOT NULL

  Таблица Thermostats:
    Id                  INTEGER PK AUTOINCREMENT
    DeviceId            INTEGER NOT NULL FK → Devices(Id)
    CurrentTemperature  REAL DEFAULT 20.0
    TargetTemperature   REAL DEFAULT 22.0
    Mode                TEXT DEFAULT 'auto'
    Humidity            REAL DEFAULT 50.0

  Таблица Lights:
    Id              INTEGER PK AUTOINCREMENT
    DeviceId        INTEGER NOT NULL FK → Devices(Id)
    Brightness      INTEGER DEFAULT 50
    Color           TEXT DEFAULT 'white'
    AutoMode        INTEGER DEFAULT 0
    ScheduledTime   INTEGER DEFAULT -1

  Таблица DoorLocks:
    Id              INTEGER PK AUTOINCREMENT
    DeviceId        INTEGER NOT NULL FK → Devices(Id)
    IsLocked        INTEGER DEFAULT 1
    AccessCode      TEXT DEFAULT '1234'
    WrongAttempts   INTEGER DEFAULT 0
    AutoLock        INTEGER DEFAULT 1
    LastAccessTime  TEXT NULL

  Таблица Cameras:
    Id              INTEGER PK AUTOINCREMENT
    DeviceId        INTEGER NOT NULL FK → Devices(Id)
    Resolution      INTEGER DEFAULT 1080
    IsRecording     INTEGER DEFAULT 0
    MotionDetection INTEGER DEFAULT 1
    StorageUsed     INTEGER DEFAULT 0
    LastMotionTime  TEXT NULL

8. СВЯЗЬ КЛАССОВ И БАЗЫ ДАННЫХ
================================================================================

При создании устройства:
  1. Создается объект C# (Например, new Thermostat(...))
  2. Сначала сохраняется в таблицу Devices (общие поля)
  3. Полученный DeviceId передается в дочернюю таблицу

При удалении устройства:
  1. По DeviceType определяется, из какой таблицы удалять
  2. DELETE из связанной таблицы (WHERE DeviceId = id)
  3. DELETE из Devices (WHERE Id = id)

При просмотре:
  1. Запрос SELECT * FROM Devices - список всех устройств
  2. При выборе → отдельный SELECT из дочерней таблицы
